    Decomposition, Abstraction and Functions

      Functions: Functions can be used to shorten a piece of code to make it look better and make it easier to organize or it can be used in a way to reorder certain codes. However, you do have to define the function first, meaning you still have to write out the code but if other people look at it, they will easily see or understand the codes.

      Decomposition: Decomposition means that you split a big question/problem and turn it into smaller questions/problems. It can be used in situations like when you're making an appliciation. You will have to split the steps of making the application down into smaller steps and then, you can assign them to people to do it.

      Abstraction: Abstraction means that you know what the code does but you don't know the code. An example is when you're using WeChat, you don't know the code but you still know what it does, so you can use it easily. This is important because abstraction make users happy and keeping users happy or satisfied is important when making your application or program.
